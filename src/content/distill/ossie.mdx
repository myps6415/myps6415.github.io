資料孤島不是新問題，但 AI 時代讓它更痛了。同樣一個營收 KPI，CRM、BI 工具、AI Agent 各寫一套，定義不一致，出來的結果擦槍走火。團隊每天花時間手動對接定義，AI 生成內容反而因為基礎邏輯不一致變得不可靠。

Apache Ossie 是 Apache 基金會新出的開源專案（以前叫 OSI），提供 JSON 跟 YAML 格式的統一語意模型規格，讓不同工具可以讀寫同一套定義。目前支援 dbt、GoodData、Polaris、Salesforce，專案裡也有完整的 TPC-DS 範例可以參考，驗證工具也準備好了。

如果你公司也有定義兜不起來的困擾——不同部門用不同工具，定義各自為政——值得把這專案加入觀察清單。先看一下 roadmap，了解未來打算支援哪些工具，也歡迎直接參與貢獻。
